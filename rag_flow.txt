📘 RAG-Based Customer Support Assistant using LangGraph & HITL
🔷 1. Introduction
1.1 What is RAG?

Retrieval-Augmented Generation (RAG) is a hybrid AI approach that combines:

Information Retrieval (search relevant documents)
Text Generation (generate answers using LLM)

Instead of relying only on pre-trained knowledge, RAG:

Fetches real-time relevant context
Generates accurate, grounded responses
1.2 Why RAG is Needed

Traditional LLMs:

❌ Hallucinate (generate incorrect facts)
❌ Lack domain-specific knowledge
❌ Cannot access private documents

RAG solves this by:

✔ Using external knowledge sources (PDFs)
✔ Improving accuracy & reliability
✔ Supporting enterprise use cases
1.3 Use Case Overview

This system acts as a Customer Support Assistant that:

Answers queries from a university policy PDF
Provides context-aware responses
Escalates to human agents when uncertain

2.3 Component Description
📄 Document Loader
Loads PDF using PyPDFLoader
Extracts structured text
✂️ Chunking Strategy
Chunk size: 800 tokens
Overlap: 100 tokens
Ensures:
Context continuity
Better retrieval
🧠 Embedding Model
Model: all-MiniLM-L6-v2
Converts text → dense vectors
🗂 Vector Store (ChromaDB)
Stores embeddings + metadata
Enables fast similarity search
🔍 Retriever
Uses cosine similarity
Retrieves Top-K relevant chunks
🤖 LLM (Groq API)
Model: LLaMA 3.1 / Mixtral
Generates contextual answers
🔄 Workflow Engine (LangGraph)
Controls execution flow
Maintains system state
🚦 Routing Layer
Decides:
Answer OR escalate
👤 HITL Module
Human fallback for:
Low confidence
Complex queries
2.4 Data Flow
📥 Ingestion Flow
PDF → Loader → Chunk → Embedding → ChromaDB
📤 Query Flow
User Query
→ Embed Query
→ Retrieve Chunks
→ LLM Generate Answer
→ Confidence Check
→ (Optional) HITL
→ Response
2.5 Technology Choices
Component	Technology	Justification

LLM	Groq (LLaMA)	Low latency
UI	Streamlit	Fast prototyping
Embedding	Sentence Transformers	Efficient
2.6 Scalability Considerations
